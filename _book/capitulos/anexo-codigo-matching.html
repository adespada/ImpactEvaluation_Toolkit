30  Código R: Matching y Propensity Score

NotaSobre este anexo

Este anexo contiene el código R completo para el taller práctico del método Matching / Propensity Score. Para la teoría y explicación del método, consulte el capítulo correspondiente.

30.1 Taller práctico

NotaSobre este taller

Vamos a trabajar con datos simulados de un programa de microfinanzas que otorgó microcréditos a 100 personas. Otras 100 personas no participaron. El resultado de interés es el ingreso mensual. Los participantes se autoseleccionaron: personas con más educación, más jóvenes, con mayor ingreso previo y de zona urbana tenían mayor probabilidad de participar.

30.1.1 Paso 1: Crear los datos simulados

library(dplyr)
library(MatchIt)
library(ggplot2)

set.seed(42)

n <- 200

# Covariables
educacion <- round(rnorm(n, mean = 10, sd = 3))
educacion <- pmax(educacion, 4)
edad <- round(rnorm(n, mean = 35, sd = 10))
edad <- pmax(edad, 18)
ingreso_previo <- round(rnorm(n, mean = 800, sd = 200))
ingreso_previo <- pmax(ingreso_previo, 300)
urbano <- rbinom(n, 1, 0.5)

# Selección al tratamiento (sesgo: más probable si joven, educado, urbano, mayor ingreso)
logit_p <- -3 + 0.15 * educacion + (-0.03) * edad +
  0.002 * ingreso_previo + 0.8 * urbano
prob_tratamiento <- plogis(logit_p)
tratamiento <- rbinom(n, 1, prob_tratamiento)

# Forzar 100 tratados y 100 controles para el ejemplo
indices_trat <- which(tratamiento == 1)
indices_ctrl <- which(tratamiento == 0)

if (length(indices_trat) > 100) {
  remover <- sample(indices_trat, length(indices_trat) - 100)
  tratamiento[remover] <- 0
} else if (length(indices_trat) < 100) {
  agregar <- sample(indices_ctrl, 100 - length(indices_trat))
  tratamiento[agregar] <- 1
}

# Resultado: ingreso mensual (efecto real del programa = 150)
efecto_real <- 150
ingreso <- 500 + 30 * educacion + (-2) * edad + 0.4 * ingreso_previo +
  100 * urbano + efecto_real * tratamiento + rnorm(n, 0, 120)

datos <- data.frame(
  id = sprintf("ID_%03d", 1:n),
  tratamiento = tratamiento,
  ingreso = round(ingreso),
  educacion = educacion,
  edad = edad,
  ingreso_previo = ingreso_previo,
  urbano = urbano
)

AdvertenciaSesgo de selección

La diferencia naive sobreestima el efecto real porque los participantes ya tenían características asociadas a mayores ingresos. Matching busca corregir este sesgo.

Nota¿Cómo leer el Love plot?

Cada punto muestra la diferencia estandarizada de medias (|SMD|) entre tratados y controles para una covariable. Los puntos azules son antes del matching y los naranjas después. Un buen matching acerca todos los puntos a cero. La línea vertical punteada (0.1) es el umbral recomendado: covariables por encima de 0.1 después del matching siguen desbalanceadas.

30.1.6 Paso 6: Estimar el ATT sobre la muestra emparejada

datos_matched <- match.data(m_out)

modelo_att <- lm(ingreso ~ tratamiento + educacion + edad +
                   ingreso_previo + urbano,
                 data = datos_matched,
                 weights = weights)

resultado <- summary(modelo_att)
coef_trat <- resultado$coefficients["tratamiento", ]

cat("========================================\n")
#> ========================================
cat("Efecto estimado (ATT):", round(coef_trat[1]), "\n")
#> Efecto estimado (ATT): 142
cat("Error estándar:", round(coef_trat[2], 1), "\n")
#> Error estándar: 20
cat("Valor p:", round(coef_trat[4], 4), "\n")
#> Valor p: 0
cat("IC 95%: [", round(coef_trat[1] - 1.96 * coef_trat[2]),
    ",", round(coef_trat[1] + 1.96 * coef_trat[2]), "]\n")
#> IC 95%: [ 103 , 181 ]
cat("Efecto real programado:", efecto_real, "\n")
#> Efecto real programado: 150
cat("========================================\n")
#> ========================================
TipInterpretación

Compare el efecto estimado con el efecto real programado (150). El matching debería acercar la estimación al valor real, corrigiendo el sesgo de la comparación naive.

30.2 Trabajando con IA paso a paso

NotaSobre esta sección

Replica los pasos del capítulo de uso de IA con prompts específicos para Propensity Score Matching sobre el dataset simulado del programa de microfinanzas. Asume que ya construyó covariables y subgrupos siguiendo el Paso 4 del capítulo general.

30.2.1 Contexto simulado

200 personas (≈100 tratadas, ≈100 control) con autoselección al programa de microcréditos. Variable de resultado: ingreso (mensual, USD). Covariables observables que predicen la participación: educacion, edad, ingreso_previo, urbano. Para los prompts asuma:

covariables <- c("educacion", "edad", "ingreso_previo", "urbano")

30.2.2 Prompt PSM.1 — Estimación del propensity score y soporte común

Actúa como analista que escribe código en R. Tengo un data frame datos con: tratamiento (1/0), ingreso, y covariables en covariables <- c("educacion","edad","ingreso_previo","urbano").

Genera un script en R que:

  1. Estime el propensity score con glm(tratamiento ~ educacion + edad + ingreso_previo + urbano, family = binomial, data = datos) y guarde datos$ps <- predict(modelo, type = "response").
  2. Grafique con ggplot2 la densidad del PS por grupo (relleno con alpha=0.5), colores #185FA5 para control y #D85A30 para tratamiento, línea vertical en min y max del soporte común.
  3. Calcule el rango de soporte común: c(max(min_trat, min_ctrl), min(max_trat, max_ctrl)) e imprima cuántos tratados quedan dentro y fuera.
  4. Cree datos_sc <- subset(datos, ps >= li & ps <= ls) con solo el soporte común.

Solo código en un bloque. Al final, en una frase, dime qué % de tratados perdidos por soporte común sería preocupante.

Cómo interpretar: si pierde más del 10–15% de tratados al imponer soporte común, los grupos son demasiado distintos y PSM puede no ser viable.

30.2.3 Prompt PSM.2 — Matching y verificación de balance

Con datos_sc (ya filtrado por soporte común) y covariables, genera un script en R que:

  1. Use MatchIt::matchit(tratamiento ~ educacion + edad + ingreso_previo + urbano, data = datos_sc, method = "nearest", caliper = 0.1).
  2. Imprima summary(m.out) y guarde el balance.
  3. Genere un Love plot con cobalt::love.plot(m.out, thresholds = c(m = 0.10)) mostrando SMD antes y después.
  4. Imprima un mensaje claro cat("Covariables con |SMD|>0.10 después del matching:", n_malas).

Solo código. Al final, en una frase, indícame qué hacer si el balance falla.

Cómo interpretar: después del matching, todas las |SMD| deben quedar por debajo de 0.10. Si alguna covariable persiste descompensada, considere agregar términos cuadráticos o interacciones al modelo de PS, o usar otro algoritmo.

30.2.4 Prompt PSM.3 — ATT con tres algoritmos (sensibilidad)

Con datos_sc y covariables, estima el ATT con tres algoritmos distintos para análisis de sensibilidad:

  1. Vecino más cercano sin reemplazo con caliper = 0.1.
  2. Vecino más cercano con reemplazo y ratio = 1.
  3. Kernel matching (puede usar Matching::Match(M=Inf, Weight=2) o ponderación por PS con WeightIt).

Para cada uno, ajusta lm(ingreso ~ tratamiento + covariables, data = matched, weights = pesos) con errores robustos. Reúne los tres efectos en una tabla con modelsummary::modelsummary() lado a lado.

Solo código. Al final, en una frase, dime qué resultado haría más creíble el ATT.

Cómo interpretar: si los tres efectos son similares y todos tienen p<0.05, el resultado es robusto. Diferencias grandes sugieren dependencia del algoritmo y debilidad del soporte común.

30.2.5 Prompt PSM.4 — Sensibilidad a confusores no observados (Rosenbaum bounds)

Con el matched dataset del Prompt PSM.2, genera un script en R que ejecute un test de sensibilidad de Rosenbaum usando rbounds::psens() o sensitivitymw, sobre los pares emparejados.

Reporte la gamma crítica: el nivel de confusor no observado (en términos de odds ratio sobre la probabilidad de tratamiento) que volvería el efecto no significativo.

Solo código. Al final, en una frase, dime cómo se interpreta gamma=1.5 vs gamma=3.0.

Cómo interpretar: gamma alto (≥ 2) significa que se necesitaría un confusor no observado muy fuerte para anular el resultado, lo que da credibilidad. Gamma bajo (≤ 1.3) sugiere que el resultado es frágil.

```{r}
  labs(x = "Propensity score estimado",
       y = "Densidad", fill = NULL,
       title = "Distribución del propensity score por grupo") +
  theme_minimal(base_size = 13) +
  theme(legend.position = "bottom",
        plot.title = element_text(face = "bold"))
```

### Paso 4: Realizar el matching con MatchIt

```{r}
#| label: matching-matchit
#| code-fold: false

m_out <- matchit(tratamiento ~ educacion + edad + ingreso_previo + urbano,
                  data = datos,
                  method = "nearest",
                  distance = "glm",
                  ratio = 1)

summary(m_out)
```

### Paso 5: Verificar balance (Love plot)

```{r}
#| label: fig-matching-love-taller
#| fig-cap: "Love plot: balance de covariables antes y después del matching"
#| code-fold: false

library(cobalt)

love.plot(m_out,
          stats         = "mean.diffs",
          abs           = TRUE,
          binary        = "std",
          un            = TRUE,                 # mostrar también ANTES del matching
          drop.distance = TRUE,                 # ocultar el propensity score
          thresholds    = c(m = 0.1),
          colors        = c("#185FA5", "#D85A30"),
          shapes        = c("circle", "triangle"),
          sample.names  = c(Unadjusted = "Antes del matching",
                            Adjusted   = "Después del matching"),
          var.names     = c(educacion      = "Educación (años)",
                            edad           = "Edad",
                            ingreso_previo = "Ingreso previo",
                            urbano         = "Zona urbana"),
          title         = "Balance de covariables (Love plot)") +
  theme_minimal(base_size = 13) +
  theme(legend.position = "bottom",
        plot.title = element_text(face = "bold"))
```

::: {.callout-note}
## ¿Cómo leer el Love plot?
Cada punto muestra la **diferencia estandarizada de medias** (|SMD|) entre tratados y controles para una covariable. Los puntos **azules** son antes del matching y los **naranjas** después. Un buen matching acerca todos los puntos a cero. La línea vertical punteada (0.1) es el umbral recomendado: covariables por encima de 0.1 después del matching siguen desbalanceadas.
:::

### Paso 6: Estimar el ATT sobre la muestra emparejada

```{r}
#| label: matching-att
#| code-fold: false

datos_matched <- match.data(m_out)

modelo_att <- lm(ingreso ~ tratamiento + educacion + edad +
                   ingreso_previo + urbano,
                 data = datos_matched,
                 weights = weights)

resultado <- summary(modelo_att)
coef_trat <- resultado$coefficients["tratamiento", ]

cat("========================================\n")
cat("Efecto estimado (ATT):", round(coef_trat[1]), "\n")
cat("Error estándar:", round(coef_trat[2], 1), "\n")
cat("Valor p:", round(coef_trat[4], 4), "\n")
cat("IC 95%: [", round(coef_trat[1] - 1.96 * coef_trat[2]),
    ",", round(coef_trat[1] + 1.96 * coef_trat[2]), "]\n")
cat("Efecto real programado:", efecto_real, "\n")
cat("========================================\n")
```

::: {.callout-tip}
## Interpretación
Compare el efecto estimado con el efecto real programado (150). El matching debería acercar la estimación al valor real, corrigiendo el sesgo de la comparación naive.
:::
