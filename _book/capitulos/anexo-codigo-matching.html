30  Código R: Matching y Propensity Score

NotaSobre este anexo

Este anexo contiene el código R completo para el taller práctico del método Matching / Propensity Score. Para la teoría y explicación del método, consulte el capítulo correspondiente.

30.1 Taller práctico

NotaSobre este taller

Vamos a trabajar con datos simulados de un programa de microfinanzas. La participación fue voluntaria: personas más educadas, más jóvenes, con más experiencia laboral, mayor ingreso previo y de zona urbana se inscribieron con mayor probabilidad. El resultado de interés es el ingreso mensual posterior al programa. Tenemos 600 personas (más controles que tratados, como suele ocurrir en programas observacionales).

30.1.1 Paso 1: Crear los datos simulados

library(dplyr)
library(MatchIt)
library(ggplot2)

set.seed(42)

n <- 600   # Pool grande con autoselección al programa

# ---- Covariables observables ----
educacion         <- pmax(round(rnorm(n, 10, 3)), 4)        # años
edad              <- pmax(round(rnorm(n, 35, 10)), 18)       # años
ingreso_previo    <- pmax(round(rnorm(n, 800, 200)), 300)    # USD
experiencia_anios <- pmax(round(rnorm(n, 8, 4)), 0)          # años
n_hijos           <- pmax(rpois(n, 2), 0)                    # conteo
urbano            <- rbinom(n, 1, 0.55)                      # binaria
casado            <- rbinom(n, 1, 0.50)                      # binaria

# ---- Autoselección al programa ----
# Coeficientes calibrados para que el ANTES tenga SMDs grandes (0.3–0.7),
# pero con suficiente solapamiento para que el matching las pueda igualar
# (DESPUÉS cerca de 0). El intercepto fija ~30% de tratados.
logit_p <- -0.9 +
  0.20 * scale(educacion)[, 1] +
  -0.25 * scale(edad)[, 1] +
  0.30 * scale(ingreso_previo)[, 1] +
  0.15 * scale(experiencia_anios)[, 1] +
  -0.20 * scale(n_hijos)[, 1] +
  0.55 * urbano +
  0.30 * casado
prob_tratamiento <- plogis(logit_p)
tratamiento <- rbinom(n, 1, prob_tratamiento)

# ---- Resultado: ingreso mensual (efecto real = 150 USD) ----
efecto_real <- 150
ingreso <- 500 +
  30 * educacion +
  -2 * edad +
  0.4 * ingreso_previo +
  20 * experiencia_anios +
  -25 * n_hijos +
  100 * urbano +
  efecto_real * tratamiento +
  rnorm(n, 0, 120)

datos <- data.frame(
  id                = sprintf("ID_%03d", 1:n),
  tratamiento       = tratamiento,
  ingreso           = round(ingreso),
  educacion         = educacion,
  edad              = edad,
  ingreso_previo    = ingreso_previo,
  experiencia_anios = experiencia_anios,
  n_hijos           = n_hijos,
  urbano            = urbano,
  casado            = casado
)

cat("Tratados:", sum(datos$tratamiento),

AdvertenciaSesgo de selección

La diferencia naive sobreestima el efecto real porque los participantes ya tenían características asociadas a mayores ingresos. Matching busca corregir este sesgo.

30.1.3 Paso 3: Estimar el propensity score

modelo_ps <- glm(tratamiento ~ educacion + edad + ingreso_previo +
                                experiencia_anios + n_hijos + urbano + casado,
                 data = datos, family = binomial)

datos$ps <- predict(modelo_ps, type = "response")

ggplot(datos, aes(x = ps, fill = factor(tratamiento,

                 method   = "nearest",
                 distance = "glm",
                 caliper  = 0.2,        # SD del logit(PS) — Austin (2011)

library(cobalt)

love.plot(m_out,
          stats         = "mean.diffs",
          abs           = TRUE,
          binary        = "std",
          un            = TRUE,                  # también ANTES del matching

  theme(legend.position  = "right",
        legend.title     = element_blank(),
        plot.title       = element_text(face = "bold"))
Nota¿Cómo leer el Love plot?

Cada punto muestra la diferencia estandarizada de medias (|SMD|) entre tratados y controles para una covariable. Los puntos azules son antes del matching y los naranjas después. Un buen matching acerca todos los puntos a cero. La línea vertical punteada (0.1) es el umbral recomendado: covariables por encima de 0.1 después del matching siguen desbalanceadas.

30.1.6 Paso 6: Estimar el ATT sobre la muestra emparejada

datos_matched <- match.data(m_out)

modelo_att <- lm(ingreso ~ tratamiento + educacion + edad + ingreso_previo +
                   experiencia_anios + n_hijos + urbano + casado,
                 data    = datos_matched,
                 weights = weights)

resultado <- summary(modelo_att)
coef_trat <- resultado$coefficients["tratamiento", ]

cat("========================================\n")
#> ========================================
cat("Efecto estimado (ATT):", round(coef_trat[1]), "\n")
#> Efecto estimado (ATT): 176
cat("Error estándar:", round(coef_trat[2], 1), "\n")
#> Error estándar: 11.6
cat("Valor p:", round(coef_trat[4], 4), "\n")
#> Valor p: 0
cat("IC 95%: [", round(coef_trat[1] - 1.96 * coef_trat[2]),
    ",", round(coef_trat[1] + 1.96 * coef_trat[2]), "]\n")
#> IC 95%: [ 153 , 198 ]
cat("Efecto real programado:", efecto_real, "\n")
#> Efecto real programado: 150
cat("========================================\n")
#> ========================================
TipInterpretación

Compare el efecto estimado con el efecto real programado (150). El matching debería acercar la estimación al valor real, corrigiendo el sesgo de la comparación naive.

30.2 Trabajando con IA paso a paso

NotaSobre esta sección

Esta sección contiene una guía completa de prompts para usar IA como asistente al preparar y aplicar Propensity Score Matching. Los prompts son genéricos: no dependen de un programa, país o sector. Reemplace lo que aparece entre corchetes [ ] por la información de su proyecto.

30.2.1 Regla general para todos los prompts

Antes de pegar cualquier prompt en una IA, agregue esta instrucción:

Actúa como analista de evaluación de impacto que escribe código en R para personas con nivel básico. Devuelve siempre:

  1. Un único bloque de código R completo y ejecutable.
  2. Comentarios breves en español antes de cada bloque.
  3. Objetos con nombres claros.
  4. Al final, un bloque cat("VERIFICACION...") con 2 o 3 cosas que debo revisar.
  5. No uses pseudocódigo.
  6. No inventes columnas que no te di.
  7. No me pidas subir la base completa. Trabaja solo con rutas locales, nombres de columnas, diagnósticos y primeras filas anonimizadas si son necesarias.

# Código R: Matching y Propensity Score {#sec-anexo-codigo-matching}

::: {.callout-note}
## Sobre este anexo
Este anexo contiene el código R completo para el taller práctico del método Matching / Propensity Score. Para la teoría y explicación del método, consulte el [capítulo correspondiente](03-matching.qmd).
:::

## Taller práctico

::: {.callout-note}
## Sobre este taller
Vamos a trabajar con datos **simulados** de un programa de microfinanzas. La participación fue **voluntaria**: personas más educadas, más jóvenes, con más experiencia laboral, mayor ingreso previo y de zona urbana se inscribieron con mayor probabilidad. El resultado de interés es el ingreso mensual posterior al programa. Tenemos 600 personas (más controles que tratados, como suele ocurrir en programas observacionales).
:::

### Paso 1: Crear los datos simulados

```{r}
#| label: matching-datos-simulados
#| code-fold: false

library(dplyr)
library(MatchIt)
library(ggplot2)

set.seed(42)

n <- 600   # Pool grande con autoselección al programa

# ---- Covariables observables ----
educacion         <- pmax(round(rnorm(n, 10, 3)), 4)        # años
edad              <- pmax(round(rnorm(n, 35, 10)), 18)       # años
ingreso_previo    <- pmax(round(rnorm(n, 800, 200)), 300)    # USD
experiencia_anios <- pmax(round(rnorm(n, 8, 4)), 0)          # años
n_hijos           <- pmax(rpois(n, 2), 0)                    # conteo
urbano            <- rbinom(n, 1, 0.55)                      # binaria
casado            <- rbinom(n, 1, 0.50)                      # binaria

# ---- Autoselección al programa ----
# Coeficientes calibrados para que el ANTES tenga SMDs grandes (0.3–0.7),
# pero con suficiente solapamiento para que el matching las pueda igualar
# (DESPUÉS cerca de 0). El intercepto fija ~30% de tratados.
logit_p <- -0.9 +
  0.20 * scale(educacion)[, 1] +
  -0.25 * scale(edad)[, 1] +
  0.30 * scale(ingreso_previo)[, 1] +
  0.15 * scale(experiencia_anios)[, 1] +
  -0.20 * scale(n_hijos)[, 1] +
  0.55 * urbano +
  0.30 * casado
prob_tratamiento <- plogis(logit_p)
tratamiento <- rbinom(n, 1, prob_tratamiento)

# ---- Resultado: ingreso mensual (efecto real = 150 USD) ----
efecto_real <- 150
ingreso <- 500 +
  30 * educacion +
  -2 * edad +
  0.4 * ingreso_previo +
  20 * experiencia_anios +
  -25 * n_hijos +
  100 * urbano +
  efecto_real * tratamiento +
  rnorm(n, 0, 120)

datos <- data.frame(
  id                = sprintf("ID_%03d", 1:n),
  tratamiento       = tratamiento,
  ingreso           = round(ingreso),
  educacion         = educacion,
  edad              = edad,
  ingreso_previo    = ingreso_previo,
  experiencia_anios = experiencia_anios,
  n_hijos           = n_hijos,
  urbano            = urbano,
  casado            = casado
)

cat("Tratados:", sum(datos$tratamiento),
    "  Controles:", sum(datos$tratamiento == 0), "\n")
head(datos, 6)
```

### Paso 2: Comparación naive (sesgada)

```{r}
#| label: matching-naive
#| code-fold: false

media_trat <- mean(datos$ingreso[datos$tratamiento == 1])
media_ctrl <- mean(datos$ingreso[datos$tratamiento == 0])

cat("Ingreso promedio - Participantes:", round(media_trat), "\n")
cat("Ingreso promedio - No participantes:", round(media_ctrl), "\n")
cat("Diferencia naive:", round(media_trat - media_ctrl), "\n")
cat("Efecto real programado:", efecto_real, "\n")
```

::: {.callout-warning}
## Sesgo de selección
La diferencia naive sobreestima el efecto real porque los participantes ya tenían características asociadas a mayores ingresos. Matching busca corregir este sesgo.
:::

### Paso 3: Estimar el propensity score

```{r}
#| label: fig-matching-ps-taller
#| fig-cap: "Distribución del propensity score por grupo"
#| code-fold: false

modelo_ps <- glm(tratamiento ~ educacion + edad + ingreso_previo +
                                experiencia_anios + n_hijos + urbano + casado,
                 data = datos, family = binomial)

datos$ps <- predict(modelo_ps, type = "response")

ggplot(datos, aes(x = ps, fill = factor(tratamiento,
                   labels = c("No participante", "Participante")))) +
  geom_density(alpha = 0.5, color = NA) +
  scale_fill_manual(values = c("No participante" = "#185FA5",
                                "Participante" = "#D85A30")) +
  labs(x = "Propensity score estimado",
       y = "Densidad", fill = NULL,
       title = "Distribución del propensity score por grupo") +
  theme_minimal(base_size = 13) +
  theme(legend.position = "bottom",
        plot.title = element_text(face = "bold"))
```

### Paso 4: Realizar el matching con MatchIt

```{r}
#| label: matching-matchit
#| code-fold: false

set.seed(2024)
m_out <- matchit(tratamiento ~ educacion + edad + ingreso_previo +
                                experiencia_anios + n_hijos + urbano + casado,
                 data     = datos,
                 method   = "nearest",
                 distance = "glm",
                 caliper  = 0.2,        # SD del logit(PS) — Austin (2011)
                 ratio    = 1,
                 estimand = "ATT")

summary(m_out, un = TRUE)
```

### Paso 5: Verificar balance (Love plot)

```{r}
#| label: fig-matching-love-taller
#| fig-cap: "Love plot: balance de covariables antes y después del matching"
#| fig-height: 5
#| code-fold: false

library(cobalt)

love.plot(m_out,
          stats         = "mean.diffs",
          abs           = TRUE,
          binary        = "std",
          un            = TRUE,                  # también ANTES del matching
          drop.distance = TRUE,                  # ocultar el propensity score
          thresholds    = c(m = 0.1),
          colors        = c("#185FA5", "#D85A30"),
          shapes        = c("triangle", "circle"),
          sample.names  = c("Antes del matching",
                            "Después del matching"),
          var.names     = c(educacion         = "Educación (años)",
                            edad              = "Edad",
                            ingreso_previo    = "Ingreso previo",
                            experiencia_anios = "Experiencia laboral",
                            n_hijos           = "N° de hijos",
                            urbano            = "Zona urbana",
                            casado            = "Casado/a"),
          title         = "Balance de covariables (Love plot)") +
  theme_minimal(base_size = 13) +
  theme(legend.position  = "right",
        legend.title     = element_blank(),
        plot.title       = element_text(face = "bold"))
```

::: {.callout-note}
## ¿Cómo leer el Love plot?
Cada punto muestra la **diferencia estandarizada de medias** (|SMD|) entre tratados y controles para una covariable. Los puntos **azules** son antes del matching y los **naranjas** después. Un buen matching acerca todos los puntos a cero. La línea vertical punteada (0.1) es el umbral recomendado: covariables por encima de 0.1 después del matching siguen desbalanceadas.
:::

### Paso 6: Estimar el ATT (= ToT) sobre la muestra emparejada

::: {.callout-note}
## Nota de notación
El parámetro que estima Matching es el **ATT** (*Average Treatment Effect on the Treated*), también llamado **ToT** (*Treatment on the Treated*). Las dos siglas se usan indistintamente en la literatura: ambas designan el efecto promedio sobre **quienes efectivamente recibieron** el programa, no sobre toda la población. Para una discusión sobre la diferencia entre ATT/ToT, ATE e ITT, consulte la sección "Efecto promedio del tratamiento" del [capítulo de introducción cuantitativa](03-intro-cuantitativa.qmd).
:::

```{r}
#| label: matching-att
#| code-fold: false

datos_matched <- match.data(m_out)

modelo_att <- lm(ingreso ~ tratamiento + educacion + edad + ingreso_previo +
                   experiencia_anios + n_hijos + urbano + casado,
                 data    = datos_matched,
                 weights = weights)

resultado <- summary(modelo_att)
coef_trat <- resultado$coefficients["tratamiento", ]

cat("========================================\n")
cat("Efecto estimado (ATT = ToT):", round(coef_trat[1]), "\n")
cat("Error estándar:", round(coef_trat[2], 1), "\n")
cat("Valor p:", round(coef_trat[4], 4), "\n")
cat("IC 95%: [", round(coef_trat[1] - 1.96 * coef_trat[2]),
    ",", round(coef_trat[1] + 1.96 * coef_trat[2]), "]\n")
cat("Efecto real programado:", efecto_real, "\n")
cat("========================================\n")
```

::: {.callout-tip}
## Interpretación
Compare el efecto estimado con el efecto real programado (150). El matching debería acercar la estimación al valor real, corrigiendo el sesgo de la comparación naive.
:::
