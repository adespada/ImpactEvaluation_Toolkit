28  Código R: Experimentos aleatorizados (RCT)

NotaSobre este anexo

Este anexo contiene el código R completo para el taller práctico del método de experimentos aleatorizados (RCT). Para la teoría y explicación del método, consulte el capítulo correspondiente.

28.1 Taller práctico

NotaSobre este taller

Se trabajará con datos simulados de una campaña de vacunación en 80 comunidades rurales. El gobierno asignó aleatoriamente 40 comunidades a recibir brigadas móviles de vacunación (tratamiento) y 40 comunidades como control. El resultado de interés es la tasa de vacunación infantil (porcentaje de niños vacunados). Además, se cuenta con características pre-tratamiento como la población, la distancia al hospital más cercano y el ingreso promedio.

28.1.1 Paso 1: Crear los datos simulados

set.seed(42)

n <- 80  # 40 tratamiento + 40 control

# Covariables pre-tratamiento
poblacion <- round(runif(n, min = 500, max = 5000))
distancia_hospital <- round(runif(n, min = 2, max = 60), 1)
ingreso_base <- round(rnorm(n, mean = 800, sd = 200))

# Asignación aleatoria al tratamiento
tratamiento <- c(rep(1, 40), rep(0, 40))

# Efecto verdadero del programa: 12 puntos porcentuales
efecto_real <- 12

# Variable de resultado: tasa de vacunación (%)
tasa_vacunacion <- 45 +
  0.003 * poblacion +
  -0.15 * distancia_hospital +
  0.01 * ingreso_base +
  efecto_real * tratamiento +
  rnorm(n, mean = 0, sd = 8)

tasa_vacunacion <- pmin(pmax(round(tasa_vacunacion, 1), 10), 98)

# Crear base de datos
datos <- data.frame(
  comunidad_id = sprintf("COM_%03d", 1:n),
  tratamiento = tratamiento,
  poblacion = poblacion,
  distancia_hospital = distancia_hospital,
  ingreso_base = ingreso_base,
  tasa_vacunacion = tasa_vacunacion
)

28.1.2 Paso 2: Tabla de balance

library(dplyr)

balance <- datos |>
  group_by(tratamiento) |>
  summarise(
    n = n(),
    poblacion_media = round(mean(poblacion), 0),
    distancia_media = round(mean(distancia_hospital), 1),
    ingreso_medio = round(mean(ingreso_base), 0),
    .groups = "drop"
  )

# Tests de diferencia de medias
test_pob <- t.test(poblacion ~ tratamiento, data = datos)
test_dist <- t.test(distancia_hospital ~ tratamiento, data = datos)
test_ing <- t.test(ingreso_base ~ tratamiento, data = datos)

tabla_balance <- data.frame(
  Variable = c("Población", "Distancia al hospital (km)", "Ingreso base (USD)"),
  Control = c(balance$poblacion_media[1],
              balance$distancia_media[1],
              balance$ingreso_medio[1]),
  Tratamiento = c(balance$poblacion_media[2],
                  balance$distancia_media[2],
                  balance$ingreso_medio[2]),
  Diferencia = c(
    round(balance$poblacion_media[2] - balance$poblacion_media[1], 1),
    round(balance$distancia_media[2] - balance$distancia_media[1], 1),
    round(balance$ingreso_medio[2] - balance$ingreso_medio[1], 1)
  ),
  Valor_p = c(
    round(test_pob$p.value, 3),
    round(test_dist$p.value, 3),
    round(test_ing$p.value, 3)
  )
)

knitr::kable(tabla_balance,
             col.names = c("Variable", "Control", "Tratamiento",
                           "Diferencia", "Valor p"))
Tip¿Cómo interpretar la tabla de balance?

Si la aleatorización funcionó correctamente, las diferencias entre grupos deben ser pequeñas y los valores p deben ser mayores a 0.05 (no significativos). Esto indica que los grupos son comparables y cualquier diferencia posterior en la tasa de vacunación puede atribuirse al programa.

Nota¿Cómo leer estos resultados?

El coeficiente de tratamiento es el efecto del programa. En nuestro ejemplo simulado, debería estar cercano a 12 puntos porcentuales (el efecto real que programamos). Note que al incluir covariables el error estándar se reduce (mayor precisión), pero el coeficiente del tratamiento no debería cambiar sustancialmente — esto es lo esperable en un RCT bien aleatorizado.

28.1.4 Paso 4: Efectos heterogéneos por subgrupos

library(dplyr)
library(ggplot2)

# Crear subgrupos por distancia al hospital
datos$subgrupo_dist <- ifelse(datos$distancia_hospital > median(datos$distancia_hospital),
                               "Lejanas (>mediana)", "Cercanas (≤mediana)")

# Crear subgrupos por tamaño de población
datos$subgrupo_pob <- ifelse(datos$poblacion > median(datos$poblacion),
                              "Grandes (>mediana)", "Pequeñas (≤mediana)")

# Estimar efectos por subgrupo de distancia
efectos_dist <- datos |>
  group_by(subgrupo_dist) |>
  summarise(
    efecto = coef(lm(tasa_vacunacion ~ tratamiento))[2],
    se = summary(lm(tasa_vacunacion ~ tratamiento))$coefficients[2, 2],
    .groups = "drop"
  ) |>
  mutate(dimension = "Distancia al hospital",
         subgrupo = subgrupo_dist)

# Estimar efectos por subgrupo de población
efectos_pob <- datos |>
  group_by(subgrupo_pob) |>
  summarise(
    efecto = coef(lm(tasa_vacunacion ~ tratamiento))[2],
    se = summary(lm(tasa_vacunacion ~ tratamiento))$coefficients[2, 2],
    .groups = "drop"
  ) |>
  mutate(dimension = "Tamaño de población",
         subgrupo = subgrupo_pob)

# Combinar
efectos_sub <- bind_rows(
  efectos_dist |> select(dimension, subgrupo, efecto, se),
  efectos_pob |> select(dimension, subgrupo, efecto, se)
)

ggplot(efectos_sub, aes(x = subgrupo, y = efecto)) +
  geom_point(size = 3, color = "#D85A30") +
  geom_errorbar(aes(ymin = efecto - 1.96 * se,
                     ymax = efecto + 1.96 * se),
                width = 0.15, color = "#D85A30", linewidth = 0.8) +
  geom_hline(yintercept = 0, linetype = "dashed", color = "gray50") +
  facet_wrap(~dimension, scales = "free_x") +
  labs(x = NULL, y = "Efecto estimado (pp)",
       title = "Efectos heterogéneos por subgrupo") +
  theme_minimal(base_size = 13) +
  theme(plot.title = element_text(face = "bold"),
        strip.text = element_text(face = "bold"))
Tip¿Cómo interpretar los efectos heterogéneos?

Si los intervalos de confianza de los subgrupos se superponen, no hay evidencia estadística de que el programa funcione de manera diferente entre subgrupos. Si un subgrupo muestra un efecto significativamente mayor, esto puede orientar la focalización de futuras intervenciones.

28.2 Trabajando con IA paso a paso

NotaSobre esta sección

Esta sección replica los pasos que vio en el capítulo de uso de IA pero con prompts específicos para RCT y aplicados a un dataset simulado sencillo. La idea es que pueda copiar cada prompt, pegarlo en su IA preferida, y obtener código listo para ejecutar. Los prompts asumen que ya pasó por los Pasos 1–4 del capítulo general (tiene base_anonima.csv limpia y los vectores covariables y subgrupos).

28.2.1 Contexto simulado

Trabajamos con la base datos del taller anterior: 80 comunidades rurales asignadas aleatoriamente (40 a brigadas móviles de vacunación, 40 a control). Variable de resultado: tasa_vacunacion. Covariables: poblacion, distancia_hospital, ingreso_base. Para los prompts de abajo asuma:

covariables <- c("poblacion", "distancia_hospital", "ingreso_base")
subgrupos   <- c("subgrupo_dist", "subgrupo_pob")

28.2.2 Prompt RCT.1 — Tabla de balance con prueba conjunta

Actúa como analista de evaluación que escribe código en R. Tengo un data frame datos con: comunidad_id, tratamiento (1/0), tasa_vacunacion, y covariables en el vector covariables <- c("poblacion","distancia_hospital","ingreso_base").

Genera un script en R que produzca:

  1. Una tabla de balance con gtsummary::tbl_summary() agrupada por tratamiento, mostrando media (DE) para cada covariable y add_p() con t-test.
  2. Una columna adicional con la diferencia estandarizada (Cohen’s d) calculada manualmente por covariable.
  3. Una prueba F conjunta que regrese tratamiento sobre todas las covariables y reporte el F y su p-valor con linearHypothesis() o equivalente.

Devuelve solo el código completo en un único bloque, sin explicaciones en prosa. Al final agrega un comentario # Verificación: con dos chequeos rápidos que yo deba hacer al ver la tabla.

Qué debe ver si la aleatorización funcionó: todos los p-valores individuales > 0.05, todas las |d| < 0.25, y la prueba F conjunta con p > 0.10.

28.2.3 Prompt RCT.2 — Atrición diferencial

Con la base datos cargada, asume que la columna respondio_endline (1/0) indica quién completó la encuesta de seguimiento. Genera un script en R que:

  1. Calcule la tasa de atrición por grupo (tratamiento == 1 vs. 0).
  2. Estime una regresión logística respondio_endline ~ tratamiento + covariables y reporte el coeficiente de tratamiento con IC95%.
  3. Compare las características de quienes respondieron vs. quienes no, dentro del grupo tratado, con tbl_summary().

Solo código. Al final, en una frase, dime cuándo la atrición sería preocupante.

Lo que verifica: que la decisión de responder no esté sistemáticamente relacionada con el tratamiento. Si el coeficiente de tratamiento es significativo, hay atrición diferencial y los resultados ITT pueden estar sesgados.

28.2.4 Prompt RCT.3 — ITT y TOT (cumplimiento imperfecto)

Con datos, asume que la columna tomo_brigada (1/0) indica cumplimiento efectivo y tratamiento indica asignación aleatoria. Genera un script en R que estime:

  1. ITT (efecto de la asignación): lm(tasa_vacunacion ~ tratamiento + covariables) con errores robustos (sandwich::vcovHC).
  2. TOT/LATE (efecto del cumplimiento) usando 2SLS con tratamiento como instrumento de tomo_brigada: AER::ivreg(tasa_vacunacion ~ tomo_brigada + covariables | tratamiento + covariables).
  3. Una tabla comparativa con modelsummary::modelsummary() con ambos modelos lado a lado.

Solo código. Al final, en una sola frase, recuérdame por qué TOT > ITT en valor absoluto cuando hay incumplimiento parcial.

28.2.5 Prompt RCT.4 — Heterogeneidad y forest plot

Con datos, covariables y subgrupos <- c("subgrupo_dist","subgrupo_pob"), genera un script en R que:

  1. Para cada subgrupo: ajuste lm(tasa_vacunacion ~ tratamiento + covariables) por categoría.
  2. Construya un data frame único con dimension | categoria | efecto | li | ls | n.
  3. Grafique un forest plot con ggplot2: barra horizontal de IC95%, una fila por categoría agrupada por dimensión, línea vertical en cero, color #D85A30 para puntos.
  4. Pruebe la interacción lm(tasa_vacunacion ~ tratamiento * subgrupo_dist + covariables) y reporte el p-valor de la interacción.

Solo código. Al final, en una frase, dime cómo interpretar barras que cruzan el cero.

28.2.7 Prompt RCT.6 — Aleatorización restringida sin perder validez

Tengo una base de línea base llamada datos_base y quiero asignar tratamiento de manera aleatoria, pero evitando diferencias iniciales muy grandes entre tratamiento y control.

Genera código en R que:

  1. Mantenga aproximadamente la mitad de la muestra en tratamiento y la mitad en control.
  2. Repita la aleatorización hasta encontrar una asignación con buen balance en estas covariables:
covariables <- c("tasa_vacunacion_base", "distancia_centro_salud",
                 "poblacion", "pobreza", "cobertura_internet")
  1. Calcule diferencias medias estandarizadas firmadas para cada covariable.
  2. Acepte una asignación si el máximo abs(SMD) está entre 0.06 y 0.12, para que el balance sea razonable sin quedar artificialmente perfecto.
  3. Guarde la semilla, el número de intentos, el máximo desbalance encontrado y una tabla de balance antes de iniciar el programa.
  4. Incluya una advertencia clara: solo se deben usar variables medidas antes del tratamiento, nunca variables de resultado posteriores.

Después del código, explica en tres frases por qué pequeñas diferencias iniciales son normales en un RCT y por qué no se debe buscar que todas las diferencias sean exactamente cero.

28.2.8 Prompt RCT.7 — Paquete de gráficos y tablas para informe

Con una base RCT llamada datos_rct, crea un script completo en R para exportar resultados listos para un artículo o informe técnico.

El script debe:

  1. Crear la carpeta plots/ si no existe y exportar en PNG:
    • rct_love_plot.png: love plot firmado en porcentaje, con puntos de tratamiento-control cercanos a cero y líneas verticales en -10%, 0 y 10%.
    • rct_outcome_distribution.png: distribución del resultado final por grupo.
    • rct_pre_post_change.png: medias pre y post por grupo con IC95%.
    • rct_itt_coefficients.png: coeficientes del efecto del tratamiento en 3 especificaciones.
    • rct_heterogeneity_forest.png: forest plot por subgrupos relevantes.
  2. Crear la carpeta tables/ si no existe y exportar en PNG:
    • tabla descriptiva de línea base por tratamiento y control;
    • tabla de balance con SMD y p-valores;
    • tabla de regresiones tipo artículo, similar a Stargazer, con errores estándar en paréntesis, estrellas de significancia, N y R2.
  3. Usar un diseño limpio: fondo blanco, tipografía legible, títulos cortos y notas metodológicas al pie.
  4. Comentar brevemente cada bloque del script.

Usa paquetes comunes (dplyr, ggplot2, modelsummary, gt, webshot2 o alternativa equivalente). Si falta algún paquete, incluye una línea opcional para instalarlo, pero no la ejecutes automáticamente.

28.3 Guía de recolección de datos

Importante¿Cuándo leer esta sección?

Lea esto antes de implementar el programa, cuando aún se encuentra en fase de planificación. La calidad de su evaluación RCT depende de un diseño riguroso y de la integridad de la asignación aleatoria.

28.3.1 ¿Qué datos necesita para un RCT?

Para llevar a cabo un experimento aleatorizado necesita como mínimo:

  1. Línea base (datos pre-tratamiento): medición de su variable de resultado y covariables antes de la asignación aleatoria.

  2. Registro de la asignación: documentación clara de qué unidades fueron asignadas a tratamiento y cuáles a control, y cómo se hizo la aleatorización.

  3. Datos de seguimiento (endline): medición de las mismas variables después de la intervención, para ambos grupos.

  4. Datos de cumplimiento: información sobre si las unidades asignadas al tratamiento efectivamente lo recibieron (y si alguna unidad de control accedió al programa).

28.3.2 Estructura de su base de datos

Su base de datos debe tener la siguiente estructura (formato de panel):

TipAsignación vs. tratamiento efectivo

Note que asignacion indica a qué grupo fue sorteada cada unidad, mientras que tratamiento_efectivo indica si realmente recibió la intervención. En el ejemplo, ID_003 fue asignada al tratamiento pero no lo recibió (incumplimiento). Registrar ambas variables es esencial para estimar el efecto ITT y el efecto LATE.

Antes de iniciar la recolección, verifique que puede responder “sí” a cada pregunta:

## Sobre este taller
Se trabajará con datos **simulados** de una campaña de vacunación en 80 comunidades rurales. El gobierno asignó aleatoriamente 40 comunidades a recibir brigadas móviles de vacunación (tratamiento) y 40 comunidades como control. El resultado de interés es la tasa de vacunación infantil (porcentaje de niños vacunados). Además, se cuenta con características pre-tratamiento como la población, la distancia al hospital más cercano y el ingreso promedio.

### Paso 1: Crear los datos simulados

```{r}
#| label: rct-datos-simulados
#| code-fold: false

set.seed(42)

n <- 80  # 40 tratamiento + 40 control

# Covariables pre-tratamiento
poblacion <- round(runif(n, min = 500, max = 5000))
distancia_hospital <- round(runif(n, min = 2, max = 60), 1)
ingreso_base <- round(rnorm(n, mean = 800, sd = 200))

# Asignación aleatoria al tratamiento
tratamiento <- c(rep(1, 40), rep(0, 40))

# Efecto verdadero del programa: 12 puntos porcentuales
efecto_real <- 12

# Variable de resultado: tasa de vacunación (%)
tasa_vacunacion <- 45 +
  0.003 * poblacion +
  -0.15 * distancia_hospital +
  0.01 * ingreso_base +
  efecto_real * tratamiento +
  rnorm(n, mean = 0, sd = 8)

tasa_vacunacion <- pmin(pmax(round(tasa_vacunacion, 1), 10), 98)

# Crear base de datos
datos <- data.frame(
  comunidad_id = sprintf("COM_%03d", 1:n),
  tratamiento = tratamiento,
  poblacion = poblacion,
  distancia_hospital = distancia_hospital,
  ingreso_base = ingreso_base,
  tasa_vacunacion = tasa_vacunacion
)

# Vista previa
head(datos, 10)
```

### Paso 2: Tabla de balance

```{r}
#| label: tbl-rct-balance
#| tbl-cap: "Tabla de balance: comparación de características pre-tratamiento entre grupos"
#| code-fold: false

library(dplyr)

balance <- datos |>
  group_by(tratamiento) |>
  summarise(
    n = n(),
    poblacion_media = round(mean(poblacion), 0),
    distancia_media = round(mean(distancia_hospital), 1),
    ingreso_medio = round(mean(ingreso_base), 0),
    .groups = "drop"

# Tests de diferencia de medias
test_pob <- t.test(poblacion ~ tratamiento, data = datos)
test_dist <- t.test(distancia_hospital ~ tratamiento, data = datos)
test_ing <- t.test(ingreso_base ~ tratamiento, data = datos)

tabla_balance <- data.frame(
  Variable = c("Población", "Distancia al hospital (km)", "Ingreso base (USD)"),
  Control = c(balance$poblacion_media[1],
              balance$distancia_media[1],
              balance$ingreso_medio[1]),
  Tratamiento = c(balance$poblacion_media[2],
                  balance$distancia_media[2],
                  balance$ingreso_medio[2]),
  Diferencia = c(
    round(balance$poblacion_media[2] - balance$poblacion_media[1], 1),
    round(balance$distancia_media[2] - balance$distancia_media[1], 1),
    round(balance$ingreso_medio[2] - balance$ingreso_medio[1], 1)
  ),
  Valor_p = c(
    round(test_pob$p.value, 3),
    round(test_dist$p.value, 3),
    round(test_ing$p.value, 3)
  )
)

knitr::kable(tabla_balance,
             col.names = c("Variable", "Control", "Tratamiento",
                           "Diferencia", "Valor p"))
```

## ¿Cómo interpretar la tabla de balance?
Si la aleatorización funcionó correctamente, las diferencias entre grupos deben ser pequeñas y los valores p deben ser **mayores a 0.05** (no significativos). Esto indica que los grupos son comparables y cualquier diferencia posterior en la tasa de vacunación puede atribuirse al programa.

### Paso 4: Efectos heterogéneos por subgrupos

```{r}
#| label: fig-rct-heterogeneos
#| fig-cap: "Efectos heterogéneos: ¿el programa funciona igual en comunidades cercanas y lejanas?"
#| code-fold: false

library(dplyr)
library(ggplot2)

# Crear subgrupos por distancia al hospital
datos$subgrupo_dist <- ifelse(datos$distancia_hospital > median(datos$distancia_hospital),
                               "Lejanas (>mediana)", "Cercanas (≤mediana)")

# Crear subgrupos por tamaño de población
datos$subgrupo_pob <- ifelse(datos$poblacion > median(datos$poblacion),
                              "Grandes (>mediana)", "Pequeñas (≤mediana)")

# Estimar efectos por subgrupo de distancia
efectos_dist <- datos |>
  group_by(subgrupo_dist) |>
  summarise(
    efecto = coef(lm(tasa_vacunacion ~ tratamiento))[2],
    se = summary(lm(tasa_vacunacion ~ tratamiento))$coefficients[2, 2],
    .groups = "drop"
  ) |>
  mutate(dimension = "Distancia al hospital",
         subgrupo = subgrupo_dist)

# Estimar efectos por subgrupo de población
efectos_pob <- datos |>
  group_by(subgrupo_pob) |>
  summarise(
    efecto = coef(lm(tasa_vacunacion ~ tratamiento))[2],
    se = summary(lm(tasa_vacunacion ~ tratamiento))$coefficients[2, 2],
    .groups = "drop"
  ) |>
  mutate(dimension = "Tamaño de población",
         subgrupo = subgrupo_pob)

# Combinar
efectos_sub <- bind_rows(
  efectos_dist |> select(dimension, subgrupo, efecto, se),
  efectos_pob |> select(dimension, subgrupo, efecto, se)
)

ggplot(efectos_sub, aes(x = subgrupo, y = efecto)) +
  geom_point(size = 3, color = "#D85A30") +
  geom_errorbar(aes(ymin = efecto - 1.96 * se,
                     ymax = efecto + 1.96 * se),
                width = 0.15, color = "#D85A30", linewidth = 0.8) +
  geom_hline(yintercept = 0, linetype = "dashed", color = "gray50") +
  facet_wrap(~dimension, scales = "free_x") +
  labs(x = NULL, y = "Efecto estimado (pp)",
       title = "Efectos heterogéneos por subgrupo") +
  theme_minimal(base_size = 13) +
  theme(plot.title = element_text(face = "bold"),
        strip.text = element_text(face = "bold"))
```

::: {.callout-tip}
## ¿Cómo interpretar los efectos heterogéneos?
Si los intervalos de confianza de los subgrupos se superponen, no hay evidencia estadística de que el programa funcione de manera diferente entre subgrupos. Si un subgrupo muestra un efecto significativamente mayor, esto puede orientar la focalización de futuras intervenciones.
:::

## Guía de recolección de datos

::: {.callout-important}
## ¿Cuándo leer esta sección?
Lea esto **antes** de implementar el programa, cuando aún se encuentra en fase de planificación. La calidad de su evaluación RCT depende de un diseño riguroso y de la integridad de la asignación aleatoria.
:::

### ¿Qué datos necesita para un RCT?

Para llevar a cabo un experimento aleatorizado necesita como mínimo:

1. **Línea base (datos pre-tratamiento)**: medición de su variable de resultado y covariables **antes** de la asignación aleatoria.

2. **Registro de la asignación**: documentación clara de qué unidades fueron asignadas a tratamiento y cuáles a control, y cómo se hizo la aleatorización.

3. **Datos de seguimiento (endline)**: medición de las mismas variables **después** de la intervención, para ambos grupos.

4. **Datos de cumplimiento**: información sobre si las unidades asignadas al tratamiento efectivamente lo recibieron (y si alguna unidad de control accedió al programa).

### Estructura de su base de datos

Su base de datos debe tener la siguiente estructura (formato de **panel**):

```{r}
#| label: tbl-rct-estructura-anexo
#| tbl-cap: "Estructura mínima de base de datos para un RCT"
#| code-fold: true
#| echo: false

ejemplo <- data.frame(
  id = c("ID_001", "ID_001", "ID_002", "ID_002", "ID_003", "ID_003"),
  periodo = c("Línea base", "Seguimiento", "Línea base", "Seguimiento",
              "Línea base", "Seguimiento"),
  asignacion = c(1, 1, 0, 0, 1, 1),
  tratamiento_efectivo = c(1, 1, 0, 0, 0, 0),
  resultado = c(42, 58, 45, 47, 38, 40),
  covariable_1 = c(2500, 2500, 1800, 1800, 3200, 3200),
  covariable_2 = c(15, 15, 8, 8, 32, 32)
)

knitr::kable(ejemplo,
             col.names = c("ID", "Periodo", "Asignación", "Tratamiento efectivo",
                           "Resultado", "Población", "Distancia (km)"))
```

::: {.callout-tip}
## Asignación vs. tratamiento efectivo
Note que `asignacion` indica a qué grupo fue sorteada cada unidad, mientras que `tratamiento_efectivo` indica si realmente recibió la intervención. En el ejemplo, ID_003 fue asignada al tratamiento pero no lo recibió (incumplimiento). Registrar ambas variables es esencial para estimar el efecto ITT y el efecto LATE.
:::

### Checklist de recolección

Antes de iniciar la recolección, verifique que puede responder "sí" a cada pregunta:

- [ ] ¿Se ha definido la unidad de aleatorización (individuo, hogar, comunidad, escuela)?
- [ ] ¿Se tiene un listado completo del universo de unidades elegibles?
- [ ] ¿Se ha recolectado la **línea base** antes de revelar la asignación?
- [ ] ¿La aleatorización fue documentada y es reproducible (semilla aleatoria guardada)?
- [ ] ¿Se cuenta con un protocolo para manejar el incumplimiento?
- [ ] ¿Se miden las mismas variables con el mismo instrumento en línea base y seguimiento?
- [ ] ¿Se tiene un identificador único para cada unidad de observación?
- [ ] ¿Se ha calculado el tamaño de muestra necesario (análisis de poder estadístico)?
- [ ] ¿Se ha registrado el estudio en un registro público (por ejemplo, AEA RCT Registry)?

## El error más común: comprometer la aleatorización
El riesgo más grave en un RCT es que la asignación aleatoria se vea comprometida — por ejemplo, si administradores del programa reasignan beneficiarios después del sorteo, o si el grupo de control accede al tratamiento. Si la aleatorización pierde integridad, el RCT pierde su principal ventaja y los resultados dejan de ser causales. **Documente y proteja el proceso de asignación con protocolos estrictos.**
