32  Código R: Regresión discontinua

NotaSobre este anexo

Este anexo contiene el código R completo para el taller práctico de Regresión Discontinua (RDD). Para la teoría y explicación del método, consulte el capítulo correspondiente.

32.1 Taller práctico

NotaSobre este taller

Vamos a trabajar con datos simulados de un programa de becas de mérito (“Beca de Excelencia”): los estudiantes con puntaje SABER 11 igual o superior a 320 acceden a beca completa para la universidad. Los que sacan menos quedan fuera del programa. El resultado de interés es la tasa de matrícula universitaria un año después del examen. Tenemos 600 estudiantes.

32.1.1 Paso 1: Crear los datos simulados

library(rdrobust)

set.seed(2024)
n <- 600

# Puntaje SABER 11 (rango realista 220-400)
puntaje <- runif(n, min = 220, max = 400)
corte   <- 320
beca    <- ifelse(puntaje >= corte, 1, 0)

# Resultado: tasa de matrícula universitaria (%)
# - sube ~10 pp por cada 100 puntos SABER (los mejores se matriculan más,
#   exista o no la beca)
# - la beca añade un salto de +18 pp en el corte
efecto_beca <- 18
matricula <- 25 + 0.10 * (puntaje - 220) +
             efecto_beca * beca + rnorm(n, sd = 6)
matricula <- pmin(pmax(matricula, 0), 100)

datos_rdd <- data.frame(
  id        = sprintf("EST_%03d", 1:n),
  puntaje   = round(puntaje, 0),
  beca      = beca,
  matricula = round(matricula, 1)
)

head(datos_rdd, 10)

library(ggplot2)

datos_rdd$grupo <- ifelse(datos_rdd$beca == 1, "Con beca", "Sin beca")

ggplot(datos_rdd, aes(x = puntaje, y = matricula, color = grupo)) +
  geom_point(alpha = 0.35, size = 1.4) +
  geom_smooth(method = "lm", se = TRUE, linewidth = 1.2) +
  geom_vline(xintercept = corte, linetype = "dashed",
             color = "#BA7517", linewidth = 0.9) +

       y = "Tasa de matrícula universitaria",
       color = NULL,
       title = "Regresión discontinua: efecto de la beca de mérito") +
  theme_minimal(base_size = 13) +
  theme(legend.position = "bottom",
        plot.title = element_text(face = "bold"))

32.2 Trabajando con IA paso a paso

NotaSobre esta sección

Replica los pasos del capítulo de uso de IA con prompts específicos para Regresión Discontinua (RDD) sobre el dataset simulado de la beca de mérito.

32.2.1 Contexto simulado

600 estudiantes con puntaje SABER 11 entre 220 y 400. Reciben beca si puntaje >= 320 (corte). Variable de resultado: matricula (tasa de matrícula universitaria, 0–100%). El supuesto clave: alrededor del corte, los estudiantes son comparables. Cualquier salto en matricula en puntaje = 320 se atribuye al programa.

32.2.2 Prompt RDD.1 — Test de manipulación (McCrary)

Actúa como analista que escribe código en R. Tengo datos_rdd con: puntaje (SABER 11, 220–400), beca (1 si puntaje>=320, 0 si no), matricula (tasa de matrícula universitaria 0–100). El corte es 320.

Genera un script en R que:

  1. Use rddensity::rddensity(X = datos_rdd$puntaje, c = 320) para el test de manipulación.
  2. Imprima el p-valor con un mensaje: cat("Test McCrary p =", round(p,3), "—", ifelse(p>0.10, "no hay evidencia de manipulación", "POSIBLE MANIPULACIÓN — RDD comprometido")).
  3. Grafique con rddensity::rdplotdensity(rdd, X) la densidad alrededor del corte.

Solo código en un bloque. Al final, en una frase, dime qué patrón visual sería sospechoso aunque el p-valor sea alto.

Cómo interpretar: p > 0.10 sugiere que no hay manipulación de la variable de asignación. Una acumulación visual a un lado del corte (incluso con p alto) sigue siendo sospechosa.

32.2.3 Prompt RDD.2 — Continuidad de covariables alrededor del corte

Con datos_rdd y un vector covariables (en el ejemplo asume covariables placeholders como edad, sexo, educacion_padres, colegio_oficial), genera un script en R que:

  1. Para cada covariable, ejecute rdrobust::rdrobust(y = datos[[var]], x = puntaje, c = 320) y guarde coef y pvalue.
  2. Construya una tabla variable | salto | p-valor | conclusión.
  3. Imprima un mensaje claro con cuántas covariables presentan saltos significativos (p<0.05).

Solo código. Al final, en una frase, dime qué hacer si una covariable salta significativamente en el corte.

Cómo interpretar: ninguna covariable debería saltar en el corte. Si lo hace, sugiere que el corte está capturando algo más que el programa, y la inferencia causal queda comprometida.

32.2.4 Prompt RDD.3 — Estimación principal con banda óptima

Con datos_rdd, genera un script en R que:

  1. Estime el efecto principal con rdrobust::rdrobust(y = matricula, x = puntaje, c = 320) (banda automática, kernel triangular, polinomio local lineal).
  2. Imprima summary() y extraiga: efecto, SE robusto, IC95%, banda óptima usada, n efectivo a cada lado.
  3. Repita con bandas alternativas: h = h_opt * 0.5, h_opt, h_opt * 1.5, y reporte los tres efectos en una tabla.
  4. Grafique con rdplot(y = matricula, x = puntaje, c = 320, kernel = "triangular").

Solo código. Al final, en una frase, dime qué esperar si el efecto cambia mucho con la banda.

Cómo interpretar: si los tres efectos son similares en signo y magnitud, el resultado es robusto. Cambios de signo o magnitud >50% sugieren que el efecto depende fuertemente de la banda y debe reportarse con cautela.

32.2.5 Prompt RDD.4 — Tests placebo en cortes ficticios

Con datos_rdd, genera un script en R que ejecute tests placebo en cortes ficticios para verificar que el efecto está concentrado en el corte real.

Pasos:

  1. Define cortes placebo: c(260, 280, 300, 340, 360, 380) (excluye el real, 320).
  2. Para cada uno, filtra los datos a un solo lado del corte real (donde no hay tratamiento real) y estima rdrobust(y, x, c = corte_placebo).
  3. Reporta los efectos placebo en una tabla corte | efecto | p-valor.
  4. Grafica los efectos placebo con ggplot2, marcando con línea horizontal punteada el efecto en el corte real.

Solo código. Al final, en una frase, dime qué patrón haría sospechar del resultado principal.

Cómo interpretar: los efectos placebo deben ser pequeños y no significativos. Si encuentra saltos similares en cortes ficticios, el efecto principal puede ser un artefacto.

32.2.6 Prompt RDD.5 — Reporte ejecutivo

Pegando los resultados principales:

32.2.7 Prompt RDD.6 — Gráficos y tablas de diagnóstico para RDD

Con una base llamada datos_rdd, una variable de resultado matricula, una variable de asignación puntaje y un punto de corte corte <- 320, genera un script en R que exporte resultados de RDD en PNG.

El script debe crear:

  1. plots/rdd_main_plot.png: gráfico de discontinuidad con puntos binned, ajuste local a cada lado del corte, IC95% y línea vertical en el corte.
  2. plots/rdd_density_test.png: gráfico/test de densidad alrededor del corte para detectar manipulación.
  3. plots/rdd_covariate_balance.png: gráfico de continuidad de covariables pre-tratamiento alrededor del corte.

  5. plots/rdd_placebo_cutoffs.png: efectos en cortes placebo.
  6. tables/rdd_regression_table.png: tabla tipo artículo con estimación principal, bandas alternativas y especificaciones de sensibilidad.
  7. tables/rdd_diagnostics_table.png: resumen de manipulación, continuidad de covariables, placebo y sensibilidad de banda.

Usa rdrobust, rddensity, ggplot2, modelsummary, gt y webshot2 o alternativas equivalentes. Después del código, escribe cuatro frases sobre cuándo un RDD es creíble y cuándo el resultado solo debe interpretarse localmente alrededor del corte.
